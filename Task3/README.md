# **Список требований к безопасности и внешним интеграциям**

## **1. Шифрование данных**
- Все данные при передаче между системами должны шифроваться с использованием протоколов **TLS/SSL**.
- Данные, хранящиеся в СУБД, также должны быть зашифрованы, особенно **чувствительные** (например, персональные данные).
- В дополнение к транспортному шифрованию, конфиденциальные данные (например, секреты) должны дополнительно шифроваться на уровне приложения: использовать алгоритмы **AES-256**, **RSA** и другие подходящие методы.
- Персональные данные не должны храниться или передаваться, **за исключением биометрических данных**, необходимых для работы домофона.

## **2. Протоколы аутентификации и авторизации**
- Использовать существующую систему **Keycloak** для централизованной аутентификации и авторизации.
- Для межсервисного взаимодействия использовать **JWT-токены**.
- Рекомендовать использование **OAuth 2.0** с JWT-токенами для делегированного доступа или **API-ключей с ограниченным доступом**.
- На уровне транспорта — возможность использования **mTLS (mutual TLS)** для двусторонней аутентификации.
- Метод авторизации должен включать комбинацию:
  - **RBAC (Role-Based Access Control)** — определение ролей,
  - **ABAC (Attribute-Based Access Control)** — привязка прав к атрибутам (например, принадлежность ресурса собственнику недвижимости).

## **3. Безопасность от атак**
- Реализовать защиту от **DDoS-атак**.
- Внедрить механизмы **Rate Limiting** (ограничения частоты запросов) для предотвращения злоупотребления API и защиты от брутфорса.

## **4. Логирование и мониторинг**
- Обязательно вести **логирование всех входящих и исходящих запросов и ответов**.
- Запрещено хранение **конфиденциальной и секретной информации** в логах.
- При возможности внедрить **SIEM-системы** для анализа логов и выявления подозрительной активности.
- Организовать централизованный сбор и анализ логов.

## **5. Взаимодействие с внешними системами**
- Модуль `smart-house-core-app` взаимодействует с сервисами провайдеров умного дома по протоколу **HTTPS**, передавая команды и получая результаты.
- Для передачи видео с домофона может использоваться **WebSocket**.
- Так как требуется **реальное время** взаимодействия, рекомендуется использовать **синхронное** взаимодействие.
- Интеграция с интеллектуальными домофонами может потребовать поддержки специализированных протоколов, таких как **SIP**.

## **6. Управление доступом и безопасностью пользователей**
- Использовать **отдельные учетные записи** с минимальным набором привилегий.
- Разграничивать доступ к ресурсам по принципу **наименьших привилегий**.
